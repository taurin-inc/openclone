저희 팀 프론트엔드 개발자 고슈가 공통 헤더(GNB) 개편기를 기술 블로그에 정리했습니다.

https://lnkd.in/gdJpWSWN

MFE 환경에서 여러 애플리케이션에 걸쳐 동작하는 공통 헤더를 어떻게 운영할지,
그리고 그 과정에서 생기는 현실적인 문제를 어떻게 풀었는지 담은 글입니다.

이 글에서는 특히 이런 문제를 다룹니다.

1. 레거시 JS 환경(Antman)과 React 앱쉘에서 헤더를 이중 관리하던 문제를, 어댑터 모듈과 런타임 로딩 방식으로 단일 관리에 가깝게 만든 과정

2. 페이지 전환 때 반복적으로 보이던 헤더 스켈레톤 깜빡임을, 쿠키 기반 스냅샷과 data attribute + CSS 방식으로 줄인 과정

3. 검색바 스크롤 모션에서 reflow를 유발하던 height 변경을 줄이고, transform 중심 구조로 바꿔 성능을 개선한 과정

무조건 가장 이상적인 구조를 택하기보다,
현재 조직 규모와 배포 환경에서 UX와 DX의 균형점을 찾는 현실적인 과정을 정리해주셨어요 :) 

MFE 환경에서 공통 컴포넌트 운영을 고민하는 분들에게 특히 추천합니다.
