YC 스타트업 3개가 Claude Code로 회사를 구축한 방법. AI first 엔지니어링 팀, 서브에이전트로 프로덕션 시스템 구축, 비개발 직군도 제품 출시.
창업자들은 이제 터미널에서 직접 제품을 출시할 수 있으며, 개발 주기를 몇 주에서 몇 시간으로 단축하고, 비기술 창업자도 첫날부터 기존 업체들과 경쟁할 수 있게 되었습니다.

이러한 변화를 실제로 보여주는 세 개의 YC 스타트업의 인터뷰입니다.

1. HumanLayer: SQL 에이전트에서 AI Native 팀으로

2025년 1분기 동안 HumanLayer 팀은 광범위한 고객 발굴을 수행하며 AI 에이전트를 구축하는 수십 개의 엔지니어링 팀과 대화했고, 그들이 고려하지 않았던 에이전틱 개발 루프의 격차가 있음을 깨달았습니다.
"모든 팀이 자체 에이전트 아키텍처를 구축했습니다 더 나은 API를 구축하는 것만으로는 안 된다는 것을 깨달았습니다. 생태계가 성숙할 수 있도록 패턴과 원칙을 확립하는 데 도움을 줘야 했습니다."
Horthy는 "12-Factor Agents: 신뢰할 수 있는 LLM 애플리케이션의 패턴"에 그들의 배움을 문서화하게 되었습니다. 4월에 발행된 이 가이드는 바이럴이 되었고, 프로덕션 에이전트 시스템 구축 경험을 종합하여 컨텍스트 엔지니어링의 모범 사례를 강조했습니다.

CodeLayer 구축: AI 우선 엔지니어링 확장
CodeLayer는 팀이 worktree와 원격 클라우드 워커를 사용하여 여러 Claude 에이전트 세션을 병렬로 실행할 수 있도록 돕습니다. 그들은 중요한 패턴을 발견했습니다. 엔지니어가 Claude Code를 마스터하면 생산성 향상이 너무 커서 진정한 도전은 조직적인 것, 즉 전체 팀에 이러한 워크플로우를 확장하는 것이 됩니다.
"팀에 AI로 작성된 코드를 출시하는 사람이 여러 명 있으면 완전히 다른 유형의 문제가 생깁니다," Horthy가 설명했습니다. "커뮤니케이션, 협업, 도구, 관리 문제입니다. 팀이 소프트웨어를 구축하는 방식에 대한 모든 것을 재연결해야 합니다."
2025년 4분기 초부터 HumanLayer는 모든 규모의 엔지니어링 팀에 이러한 도구와 워크플로우를 배포하기 위해 여러 대형 파일럿을 체결했으며, 모두 Claude Code로 구축되었습니다.


2. Ambral: 서브에이전트로 1인 100계정 관리

B2B 스타트업의 숙제입니다. 스케일을 키우면 고객 경험이 보통 안 좋아져요. 어카운트 매니저 한 명이 50~100개의 계정을 관리하는 상황에서 "1/50의 관심으로는 제대로 된 관리가 불가능하다"고 Stettner는 말합니다.
Ambral은 이 문제를 서브에이전트로 풀었습니다. 흩어진 고객 정보(Slack, 미팅, 로그)를 종합해 각 계정의 AI 모델을 만들고, 이탈 징후를 포착하거나 확장을 제안합니다.
고객 활동과 상호작용에서 신호를 종합하여 모든 계정의 AI 기반 모델을 만듭니다. 시스템은 누가 관심이 필요하고 왜 그런지 정확히 짚어내며, 확장을 자율적으로 추진하거나 권장하면서 이탈을 방지하기 위해 불만족의 초기 징후를 포착합니다.

Stettner의 워크플로우는 명확합니다.
1.  Research (Opus): 코딩 전, 깊이 있는 리서치를 수행합니다. "계획 전에 리서치해서 많은 컨텍스트를 읽기"
2.  Plan (Opus): 구현을 위한 상세한 마크다운 계획을 짭니다.
3.  Implement (Sonnet): 계획을 바탕으로 Sonnet이 코드를 짭니다.

이 아키텍처는 제품에도 그대로 녹아들었습니다. Ambral의 엔진은 각 데이터 타입별로 서브에이전트들이 협업합니다.


3. Vulcan Technologies: 비개발자도 정부 계약을 따낸다

"전체 프로토타입을 Claude로 만들었습니다. 코딩을 전혀 모르는 상태에서요."
Vulcan의 창업자들은 엔지니어링 배경이 없었습니다. 하지만 400년 된 버지니아주의 복잡한 규제를 AI로 분석하는 프로토타입을 만들어냈고, 쟁쟁한 컨설팅 회사들을 제치고 계약을 따냈습니다.

언어가 곧 프로그래밍이다
CEO Tanner Jones는 말합니다.
"언어를 이해하고 논리적으로 생각할 수 있다면, 당신도 Claude Code를 잘 쓸 수 있습니다."
이제 그들은 Google 출신 CTO와 함께 Claude Code로 무장하고, 주 정부와 연방 정부의 규제 혁신을 이끌고 있습니다.


YC 창업자들이 전하는 Claude Code 꿀팁

1.  Claude가 연구하면서 계획하면서 구현하게 하지 마세요. 별도의 프롬프트를 사용하고 각각을 별도의 세션에서 실행하세요.
2.  컨텍스트는 신중하게 관리하기: 프롬프트에 모순이 있으면 결과물도 망가집니다. 시스템 프롬프트와 컨텍스트 관리에 목숨을 거세요.
3.  AI가 길을 잃으면 하면 바로 끊으세요. 초반 몇 턴에서 포착하면 Claude Code가 전체 잘못된 접근 방식을 완료하도록 두는 것보다 훨씬 더 많은 시간을 절약합니다
