지난 주에 가장 즐거웠던 미팅은 DM 주신 모 상장사 이사님과 즐거운 저녁자리! 엔지니어 출신이셔서 AI를 깊게 이해하고 계셨고 덕분에 티키타카가 잘돼서 재미있었다.

1. CLI 툴 중에 딱 하나만 쓴다면? Claude Code. 모델 성능 때문이 아니라 마개조하기 가장 쉬운 툴이기 때문. 온갖 종류의 컨텍스트 매니징은 물론, 심지어 시스템 프롬프트도 바꿀 수 있다. 자유도가 가장 높다
2. 왜 커서 같은 IDE에서 Claude Code 안 쓰고 터미널에서 쓰는지? IDE 는 내가 직접 코드를 보게 된다. 그런데 그러면 코드를 리뷰하고 싶어지고 급 느려진다. 어차피 1년 뒤에 나오는 모델은 너무 똑똑해서 AI가 쓴 코드를 보는게 의미가 없을텐데. 코드 대신 한 단계 추상화해서 개발할 수 있는 구조에 익숙해져야 한다.
3. AI가 틀리면 어떻게 하지? 멀티턴으로 대화하면서 다시 시키는게 아니라 실패 보고서를 쓰고. /clear -> 이전 요청 + 실패 보고서를 같이 주입. 왜? 어텐션을 생각해보면, 좋은 컨텍스트만 신중하게 늘려야함. 안 그러면 중요한 컨텍스트에 적게 어텐션한다.
4. bypass permissions on 모드를 디폴트로 사용하는게 핵심일 수도 있다. 왜냐면 권한을 많이 줄수록 컨텍스트를 잘 주는 레버리지가 커지기 때문
5. 멀티턴으로 이미지를 조작했을 때 맥락이 유지되는 건 디퓨전 모델보다는 오토리그레시브한 모델에서 더 잘한다. gpt-4o가 그랬다. 왜? 이미지는 2차원 픽셀이라고 볼 수 있지만 그걸 한 줄씩 쭈욱 펴면 텍스트랑 똑같으니까
6. AI에 확신을 갖고 베팅하는 멘탈 모델이 도움이 된다. 무조건 AI로 다 할 수 있다는 믿음이 있어야 작은 부분이라도 AI로 대체할 수 있는 부분을 찾는다. techno optimist가 돼자
7. 프로덕트 오버행. 새로운 모델이 나오면 그 전엔 성능이 안 좋아서 쓸모 없었던 게 갑자기 효용이 생긴다. ChatGPT, Claude Code가 그렇게 탄생했다. 즉, 새로운 모델이 나올 때마다 내 워크플로우가 바뀌어야 한다.
8. 클로드코드로 딥리서치를 대체할 수 있을까? 8-90퍼센트 정도일듯. 클로드코드로 프롬프트를 잘 주면 조사 정리 작성할 수 있지만 딥리서치는 긴 출력에 최적화된 모델을 사용한다고 들음
