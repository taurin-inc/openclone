Claude Code로 작업할 때면 워크플로우까지 같이 수정하곤 합니다.
오늘 3단계 메타 워크플로우를 어떻게 했는지 공유할게요.
스킬 사용하면서 대화 → 스킬의 워크플로우 업데이트 → 스킬 리팩토링입니다.

43분짜리 a16z Consumer AI 리뷰 영상을 받았습니다.
Claude Code에 링크만 던지고 "digest"라고 쳤어요.

저는 영상을 안 봤어요. 그냥 문제부터 풀었습니다.

1. [1차 퀴즈 결과: 6/9 (67%)]

틀린 문제들이 재밌었어요.
"2025년 바이럴을 주도한 건 코딩 도구다" - 틀렸습니다. 이미지/비디오였어요.
"소셜 AI 실패 원인은 비디오 품질" - 틀렸습니다. Status Game 부재였어요.
"Grok의 경쟁 우위는 X 플랫폼 독점" - 틀렸습니다. 컴퓨트 무병목이었어요.

찍어서 틀린 게 아니에요.
확신을 갖고 틀렸습니다.
이게 핵심이에요. 모르는 걸 모른다고 인지하게 됩니다.

틀린 부분만 요약과 전체 스크립트에서 다시 찾아봤어요.
"한 번 더 퀴즈" 눌러서 2차 도전.
7/9 (78%)로 올랐습니다.

[2. 여기서 워크플로우를 하나 개선했습니다.]
"퀴즈에서 틀린 내용을 문서에 저장해줘"

그랬더니 "핵심 학습 포인트" 섹션이 추가됐어요.
제가 틀렸던 5가지가 직접 인용과 함께 정리되어 있습니다.

[3. 이게 제 YouTube digest를 예시로 한 메타 워크플로우입니다.]

1. 퀴즈 먼저 - 내용 보기 전에 능동적 사고
2. 틀린 부분 복습 - 요약/스크립트에서 확인, Deep Research도 가능
3. 불편하면 즉시 개선 - "이것도 해줘" 한 마디로 워크플로우 업데이트
4. 형식도 개선 - /plugin-dev:skill-development로 스킬 리팩토링

43분 영상을 수동적으로 보는 대신, 퀴즈 풀면서 30분 만에 핵심 인사이트 5개를 건졌습니다.

최근엔 이렇게 세션의 메타적인 정보를 자동으로 /wrap 해주는 커맨드를 쓰고 있는데, 좀 더 깎아서 다음엔 이걸 보여드리겠습니다.
