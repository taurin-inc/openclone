PM이 알아야 할 AI 지식: Tool Call, Agent, MCP, GPT-5 (심화: Kimi K2)

1. Tool Call이란?
Tool Call을 지원하는 모델은 스스로 부족한 정보나 기능이 있을 때 데이터베이스를 질의하거나 실시간 정보를 가져오고, 코드 실행같이 복잡한 작업을 외부 도구를 통해 수행할 수 있습니다.
예를 들어 "오늘 서울 날씨는?"라는 질문을 받으면, 사전 학습한 지식에는 없는 정보이기 때문에 날씨 API를 호출해서 답변을 생성합니다. 

2. 대표적인 Tool Call Agent: Deep Research
에이전트는 능동적으로 자율적으로 목표를 설정하고 계획을 세우며 툴을 활용해 행동까지 수행합니다.(프로액티브)
이런 구조를 가장 잘 보여 주는 사례가 Deep Research 모드입니다. Deep Research는 한 번의 요청으로 "검색-읽기-계획 수정" 루프를 수십 번 반복하여, 수백 개의 출처를 읽고 핵심만 추려 종합 보고서를 만듭니다. 덕분에 사용자는 몇 시간 걸리던 리서치 업무를 몇 분 만에 끝낼 수 있습니다. Tool Call 중심 액션 루프가 얼마나 강력한지 처음 딥리서치를 사용했던 쾌감을 잊을 수가 없습니다.

3. 코딩 영역에서 Tool Call이 만들어 낸 성능
마찬가지로 개발할 때에는 코딩-실행-디버그를 반복해야 합니다. Claude Code는 LLM이 로컬 Shell을 직접 제어해 코딩은 물론, 빌드, 테스트까지 실행하며 목표를 달성할 때까지 동작합니다. 마지막에는 PR까지 만들어주죠. 
기존 코딩 모델이 좋은 "코드"만 작성했다면, 툴 콜링까지 하는 LLM 클라이언트는 생산성이 말도 안되게 높아집니다. 그리고 이 Tool Call 을 현 시점에서 기가 막히게 해내는 모델이 클로드이고, 그 클로드의 토큰을 극한까지 쓰게 해주는게 Claude Code 입니다. 그래서 개발자들이 열광하는 것입니다.

4. Tool 생태계를 만든 MCP
Anthropic이 2024년 11월 공개한 MCP는 Tool Call을 위한 표준 인터페이스입니다.
- AI 앱에 데이터 소스를 연결하려면 매번 포맷을 맞춰야했던 과거와 달리, MCP 클라이언트-서버를 한 번씩만 구현하면 어디서나 호환됩니다.
- tool_use / tool_result 블록으로 원격 툴 호출을 API 수준에서 정식 지원하므로, 별도 SDK 없이도 동작합니다.
- 개발자가 기여하기 좋은 생태계: 오픈소스를 중심으로 다양한 MCP 서버가 공개됐고 새로운 툴에 영감을 받아서 또 새로운 툴이 만들어졌습니다. Cursor, Replit 같은 클라이언트들은 mcp를 빠르게 채택하며 성능에서 기인한 매출 이득을 많이 봤습니다. "진짜로 1시간 만에 앱이 다 만들어져요"

5. GPT-5: 툴 콜 벤치마크를 다시 써 버리다
τ²-Bench Telecom 97%: GPT-5는 실제 고객센터처럼 상담사(에이전트)와 고객이 각각 자기 쪽에서 필요한 조작을 해나가는 ‘dual-control’ 시험에서 평가됐습니다. 
이 벤치마크는 “휴대폰 재부팅·설정 켜기/끄기(고객)”와 “회선/요금제 확인·변경(상담사)” 같은 일을 30단계 이상, 순서대로 처리하면서 동시에 병행 작업도 수행해야 완결되는 복잡 시나리오로 구성됩니다. GPT-5는 이 긴 절차를 처음부터 끝까지 안정적으로 마쳐, 기존 최고 성능(예: Claude 3.7 ≈ 49%, Kimi K2 ≈ 66%)을 거의 두 배 가까이 끌어올린 97%에 도달했습니다.

6. (심화편) Kimi K2 – Tool Call 을 잘하기 위해 없는 데이터도 만듭니다.
이렇게 툴이 중요하다 보니 모델의 학습 단계에서부터 Tool Call 하는 과정을 학습하기 시작했습니다. 최근 공개된 Kimi K2 모델은 실제 오픈소스 MCP 3000개를 분석해 2만 개 이상 가상 툴을 합성 생성했습니다. 
무슨 말이냐면 세상에 존재하는 MCP는 수가 너무 적고, 그마저도 개발 쪽 툴에 치중돼있었습니다. 그래서 모델이 이 데이터만 학습할 경우에는 좋은 성능을 내지 못하니, LLM으로 그럴듯한 가짜 툴을 생성해서 그걸로 학습을 시켰다는 뜻입니다. 이 아이디어는 데이터 합성 쪽에서는 흔한 일이기는 하지만 Tool Call을 위해 돌렸다는게 인상적이에요
그 결과 Kimi K2는 공개 벤치마크에서 Claude 4에 견줄 만큼 높은 Tool Call 성공률을 보였습니다.

Key Takeaways
- Tool Call 은 AI에게 말 그대로 강력한 도구를 쥐어줍니다.
- Deep Research 를 실행할 때 화면을 자세히 보세요. 에이전트가 어떻게 루프를 돌며 Tool Call하는지 볼 수 있습니다.
- Claude Code 가 좋은 이유는 Tool Call 이 코딩에서 너무나 효과적이기 때문입니다.
- MCP 덕분에 툴이 대폭발(?)했습니다.
- GPT-5도 코딩 쪽 성능을 강조할 때 툴 콜 성능을 강조합니다.
- Kimi K2 는 합성 데이터까지 동원해서 Tool Call 을 학습했습니다.
