Claude Code를 만드는 팀이 이틀 만에 20개 프로토타입을 만든 비밀. Anthropic이 LLM 경쟁에서 이기기 위해서 공개하지 않으려고도 했던 Claude Code의 시작은 의외로 소박했습니다. 보리스의 개인 프로젝트에서 출발했죠. 

여기에 파일 시스템과 배치(batch) 명령어를 실행할 수 있는 도구를 붙였을 때, 단순한 실험은 곧 엔지니어들에게 매혹적인 무언가로 바뀌었습니다. 단순히 텍스트를 다루는 수준이 아니라, 운영체제 터미널에 명령을 내리고 실제 프로그램을 실행하며, 폴더 이동과 파일 복사까지 가능해진 것입니다.

Cursor 같은 IDE와 달리, cd, ls 같은 명령어들이 중요한 의미를 갖는 CLI 환경에서 Claude Code는 자신만의 색깔을 드러내기 시작했습니다.

1. 제품 오버행(Product Overhang)

AI 모델이 이미 특정 능력을 갖추고 있지만, 그 능력을 제품으로 구현한 사례가 없는 상태를 제품 오버행이라고 부릅니다. Claude Code는 바로 이 오버행을 현실로 끌어낸 대표적인 사례입니다.

(OpenAI 의 PM도 ChatGPT에 대해 이 개념을 똑같이 인용한 적이 있었는데 흥미롭습니다.)

흥미로운 점은 처음에는 외부 공개조차 망설였다는 사실입니다. 이 제품이 Anthropic 내부에서만 사용하다가, 불과 닷새 만에 엔지니어링 팀 절반 이상에게 퍼지는 것을 보고는 “이건 외부에서도 성공할 수 있겠다”는 확신을 얻었다고 합니다.


2. 데이터 사이언티스트들의 자발적 사용

Claude Code는 엔지니어만을 위한 도구가 아니었습니다. 어느 날 보리스가 사무실을 지나가다 보니 데이터 과학자들 역시 Claude Code를 실행하고 있었습니다. 쿼리를 작성하고, 시각화를 만들고, 심지어 여러 인스턴스를 동시에 돌리기도 했죠. 자연스럽게 확산된 활용은 이 툴의 잠재력을 보여주는 사례였습니다.


3. 익숙한 스택 + 단순한 아키텍처

Claude Code는 TypeScript와 React로 만들어졌습니다. 이유는 단순합니다. Claude가 이미 잘하는 스택을 선택했기 때문이죠. 만약 낯선 스택을 선택했다면 추가적인 학습이 필요했을 겁니다. 하지만 “온 디스트리뷰션(on distribution)” 영역에서 개발했기에, Claude는 스스로 코드를 작성하며 성능을 극대화했습니다. 실제로 현재 코드의 약 90%는 Claude가 직접 작성했다고 합니다.

아키텍처 또한 가능한 한 단순하게 설계되었습니다. Claude 모델 위에 올려진 가벼운 셸(shell)에 불과합니다. 새로운 모델이 나올 때마다 불필요한 코드와 도구를 과감히 삭제하고, 사람들이 모델을 “날 것(raw)” 그대로 경험할 수 있도록 했습니다.

터미널 위에서 운영되는 툴이 TypeScript와 React로 만들어졌다는 것은 놀라운데요. Ink 패키지는 사실 알고 보면 OpenAI Codex CLI, Gemini CLI 등등 우리가 알고 있는 모든 종류의 CLI 코딩 에이전트가 사용하고 있습니다. On distribution의 힘은 생각보다 강력합니다.

3. 폭발적인 속도와 실험

Anthropic 팀은 여름 동안 하루 평균 5개의 PR을 머지했고, 하루 한 번은 외부 배포를 진행했습니다. 기능 개발과 배포 속도는 대부분의 기술 회사가 상상하기 어려운 수준이었습니다.

대표적인 사례가 바로 이틀 만에 20개의 프로토타입을 만든 “할 일 목록(todo lists)” 기능입니다. 보리스는 직접 Claude와 상호작용하며 작은 수정과 반복을 거듭했고, 동료 피드백을 빠르게 반영했습니다. 마음에 안 들면 즉시 새로운 프롬프트로 다시 시도했죠.

4. AI가 담당하는 개발 프로세스

Claude Code 팀은 다른 엔지니어링 팀과는 전혀 다른 방식으로 일합니다. Claude가 기존 엔지니어가 맡던 많은 일을 대신하기 때문입니다.
	•	코드 리뷰 1차: Claude가 모든 PR을 먼저 검토
	•	보안 리뷰: Claude가 1차 리뷰 후, 필요 시 사람 검토
	•	테스트 작성: 거의 전적으로 Claude가 작성
	•	GitHub 이슈 triage 및 구현 시도: 작은 버그 픽스는 20~40%를 한 번에 해결

높아진 생산성 덕분에 테스트 주도 개발(TDD)이 다시 부활했습니다. 모델이 테스트를 먼저 작성하고, 그 테스트를 통과하는 코드를 구현하는 방식은 오히려 과거보다 훨씬 효율적으로 작동했습니다.

5. 서브에이전트는 단 3일만에 만들어졌다. 

Anthropic 팀은 “멀티 에이전트 아키텍처가 꼭 유익하다는 증거가 부족하다”는 점을 받아들이며, Claude Code를 가능한 한 단순하게 유지했습니다. 서브에이전트 기능조차 단 3일 만에 구현했는데, 핵심은 새로운 발명을 하기보다 이미 존재하는 방식(슬래시 명령어, .md 파일)을 재활용하는 것이었습니다. 유저가 익숙하게 사용하는 툴을 그대로 제공할 수 있었기 때문입니다.

6. AI 엔지니어링 팀의 미래

오늘날 Anthropic의 기술 인력 약 80%가 매일 Claude Code를 사용합니다. 엔지니어링 워크플로우 전반에 녹아든 AI는 이제 단순한 보조가 아니라, 팀의 핵심 구성원으로 기능하고 있습니다.

보리스가 강조하듯, “제품 마인드를 가진 엔지니어 + AI 도구”의 조합은 새로운 기회를 창출합니다. 회의보다 프로토타입을, 복잡함보다 단순함을 추구하는 태도가 Claude Code의 압도적인 속도와 혁신을 가능하게 한 비밀이었습니다.
