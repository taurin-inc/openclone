LG AI 리서치에서 Claude Code 세미나를 했습니다. 두 시간 세미나 중에 핵심만 정리했습니다. "바이브코딩을 잘 하려면 Context를 관리하는 시스템이 필요합니다. 알고보면 Claude Code 는 정교한 Context 관리 도구거든요."

Context는 쉽게 말해서 프롬프트를 입력하면 그 앞(혹은 뒤)에 붙는 텍스트 덩어리라고 할 수 있습니다. LLM에게 "메뉴 버튼을 오른쪽으로 옮겨"라고 하면 무슨 말인지 못 알아듣지만, 메뉴 버튼에 대한 코드와 메뉴가 뜻하는 바를 먼저 써주고 옮겨달라고 하면 그때 알잘딱 합니다.

클로드코드에서 /context 명령어를 입력하면 힌트를 얻을 수 있는데, 내가 프롬프트를 입력하기 전에 굉장히 많은 텍스트가 붙어있습니다. 순서대로 시스템 프롬프트, 툴(기본 + mcp), 서브 에이전트, Claude.md 다음에 메시지가 옵니다.

1. 시스템 프롬프트
"The assistant is Claude, created by Anthropic."로 시작하고 LLM에게 기본적인 행동 가이드를 제공합니다. LLM에게 정체성을 심고, 하지 말아야 하는 일을 가이드합니다. 흥미롭게도 Claude Docs 의 Release Notes 를 통해 클로드 앱 시스템 프롬프트를 공개하고 있습니다. https://lnkd.in/g3ZzMF87 

2. 툴(기본 + MCP)
클로드 코드가 기본적으로 제공하는 File Read, Write, Bash, WebSearch, WebFetch 툴의 설명이 있어서 사용자 요청을 해결하기 위해서 웹 검색을 해야 판단하면 여기서 도구를 찾을 수 있습니다. 마찬가지로 MCP를 추가하면 클로드가 선택할 수 있는 도구가 늘어납니다. MCP를 많이 붙일수록 딱 맞는 툴 선택이 어려워지기 때문에 신중하게 고르는 것이 더 좋습니다. 그리고 특정 MCP를 꼭 사용하길 바란다면 해당 mcp 가 어떤 툴을 가지고 있는지 조회해서 툴의 이름을 정확히 지시하는 것이 좋습니다.

3. 서브 에이전트
미리 서브에이전트를 생성해뒀다면 내 요청에 맞는 서브에이전트를 알잘딱으로 호출하는데, 그건 서브에이전트의 name, description이 항상 Context로 들어가기 때문입니다. 코드 리뷰 서브에이전트를 운영하고 있다면 단순히 코드를 리뷰해달라는 요청을 했을 때 그 에이전트를 사용하는 건 이런 원리입니다. 툴을 고르는 것과 동일한 매커니즘입니다.

주목할 점은 서브 에이전트는 전체 세션과는 별도의 컨텍스트 창을 갖는다는 것입니다. 서브 에이전트 안에서 수십개의 파일을 읽어도 그만큼의 컨텍스트가 소비되는 것이 아닙니다. 서브에이전트가 동작을 모두 끝낸 뒤 출력하는 1-2k 만큼의 토큰만 컨텍스트에 포함됩니다.

이런 원리를 이용해서 planning 하는 서브에이전트를 사용하면 굉장히 좋습니다. 코드베이스와 웹을 두루두루 탐색하면서 작업의 플랜을 아웃풋으로 내는 에이전트는 성능과 컨텍스트 윈도우 효율을 모두 잡습니다. 모든 파일이 이번 작업에 필요한 것은 아니니까 꼭 필요한 파일을 읽은 정보만 추려준다면 분명히 도움이 됩니다.

4. CLAUDE.md
흔히 CLAUDE.md 가 시스템 프롬프트 바로 뒤에 붙을 것이라고 예상하는데 그렇지 않습니다. 메모리 파일이라고 칭하는 이 파일은 실은 메시지 바로 앞에 붙습니다. 클로드에서 중요한 지시는 마지막에 내리라는 가이드를 준 것도 보면 분명히 의미하는 바가 있습니다. 꽤 중요하게 읽는다는 것이죠.
이 파일에는 보통 클로드가 자주 실행할 빌드, 실행 명령어나 파일 구조, 개발 동작 가이드가 들어있습니다.

이번 Claude code 업데이트를 통해 Reserved 가 추가됐는데요. autocompact, output tokens 를 위해 미리 컨텍스트를 확보해놓는 것으로 보입니다. 이 글을 올리는 시간을 기준으로 아직 공식 설명이 안 나왔고, 이것에 대해 github issue 도 제기돼있는 상태입니다. 

Claude Code 의 핵심 기능은 Context를 관리하고 변경하는 것입니다. 이런 원리를 이해하고 각 기능을 적절히 이용하면 훨씬 수준 높은 코드 결과물을 얻을 수 있게 됩니다. 

패스트캠퍼스를 통해 강의를 촬영하고 있는데 이런 내용을 영상으로 담았습니다. 글로는 쉽게 전달하기 어려운지라 실제로 Claude Code를 실행하면서 설명할 수 있는 기회라 추천드립니다. 강의는 시간을 돈으로 사는 것이라고 생각하는데요. 구매하신다면 돈이 아깝지 않게 최선을 다 하겠습니다.
