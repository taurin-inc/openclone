소스 URL 두 개 → 10분짜리 팟캐스트 → YouTube 업로드. "/podcast" 스킬(플러그인)을 만들어서 사용 중입니다. 이 팟캐스트를 듣는 사람은 세상에 저 하나뿐이에요. 읽고 싶은 글을 나만의 음성 콘텐츠로 바꾸는 파이프라인입니다.

오늘 만든 에피소드의 소스는 두 개입니다. Y Combinator CEO Garry Tan이 공개한 "gstack"이라는 Claude Code 설정, 그리고 변호사 Zack Shapiro가 쓴 "The 10x Lawyer" 에세이. 언뜻 관련 없어 보이는 두 글인데, Claude Code에게 던졌더니 "판단 인프라"라는 공통 키워드를 찾아냈어요. Shapiro의 핵심: 소프트웨어에는 "10배 프로그래머"가 있었는데 법률에는 "10배 변호사"가 없었다 — 재능의 차이가 아니라 인지 대역폭과 위임 비용이라는 구조적 천장 때문이다. AI가 이 천장을 제거하면서 처음으로 한 명이 다섯 명 몫을 해냅니다. Garry Tan의 gstack은 이걸 코드로 보여줘요. 6개의 스킬이 각각 다른 인지 모드를 담당합니다 — CEO 리뷰는 제품 감각, 엔지니어링 리뷰는 아키텍처, ship은 테스트부터 PR까지 원클릭 배포. 이걸로 7일에 100개 PR을 쳤습니다.

두 글의 교차점이 팟캐스트 3부의 핵심이에요. "래퍼는 함정이다." AI를 블랙박스로 쓰면 역량이 위축되고, 도구로 직접 다루면 역량이 복리로 성장한다. Shapiro는 이걸 "바벨 효과"라고 불렀습니다. 파이프라인은 6단계예요. (1) 소스 수집 — URL을 던지면 병렬 분석, GitHub 레포는 클론해서 구조까지 봄. (2) 스크립트 — 3600자 한국어 구어체, 숫자는 한글로, 영어 이름은 발음으로. (3) TTS — OpenAI gpt-4o-mini-tts, marin 보이스, 1500자씩 청크 분할 후 ffmpeg 병합. (4) MP4 — 다크 배경에 한글 제목 오버레이. (5) YouTube OAuth 업로드. 파이프라인 전체를 Claude Code 플러그인으로 오픈소스 공개했고, Python 표준 라이브러리와 OpenAI API 키, ffmpeg만 있으면 됩니다. 나 혼자만을 위한 제품 만들기, 콘텐츠 만들기 — 뭐든 할 수 있어요.
