Claude Code 팀은 단 20분만에 새로운 기능을 만들어냅니다. 그것도 유저와 대화하면서요. 이 팀은 클로드코드를 써서 클로드코드를 만들기로 유명한데요. 유저가 클로드 코드의 서브 에이전트와 훅을 연동시켜달라고 트위터에서 이야기를 하자마자 다음 버전 배포에 추가했습니다.

이런 말도 안되는 속도 뒤에는 엄청난 워크플로우가 숨겨져 있겠죠?
1. Feature Request 메타 프롬프팅
2. 코드 리뷰, 리팩토링
3. 유닛 테스트, E2E 테스트
...
이런 것들이 Skill, Sub Agent, Commands로 구성돼 있을거에요. 그리고 팀 단위, Anthropic 전사 단위에서 사용하기 위해 plugin 으로 관리할 겁니다. 

Claude Code 처럼 많은 유저가 사용하는 프로덕트가 단 20분만에 기능을 개발하고 다음 버전 배포에 추가하다니 배울 점이 많습니다. 

Boris Cherny (Claude Code 개발자)
"1년 전, Claude는 이스케이프 문제 없이 bash 명령어를 생성하는 데도 어려움을 겪었습니다. 몇 초 또는 몇 분 단위로 작동했죠. 우리는 언젠가 코딩에 광범위하게 유용해질 수 있다는 초기 신호를 봤습니다.
오늘날로 돌아오면, 지난 30일 동안 저는 259개의 PR을 머지했습니다 -- 497개의 커밋, 4만 줄 추가, 3만 8천 줄 삭제. 모든 한 줄 한 줄이 Claude Code + Opus 4.5로 작성되었습니다. Claude는 (Stop 훅을 사용하여) 몇 분, 몇 시간, 며칠 동안 일관되게 실행됩니다. 소프트웨어 엔지니어링이 변화하고 있으며, 우리는 코딩 역사의 새로운 시대에 진입하고 있습니다. 그리고 우리는 아직 시작 단계에 불과합니다..."
