하루 날 잡아서 워크플로우를 AI Native 하게 뿌수고 왔습니다. Pre Sales 단계에서 클라이언트의 진짜 요구사항을 발굴하는 건 정말 중요한데요. 이 단계를 Claude Code 와 Airtable로 프로토타이핑했습니다. 레슨런을 공유합니다.

클라이언트와 미팅할 때 AI Note Taker 는 필수
인핸스는 클라이언트와의 회의록이 정말 잘 보존돼있었어요. 이 회의록을 AI가 요약하고, 이 맥락을 주입하면 사람이 만든 것과 거의 유사한 결과가 나왔습니다. "이 정도는 AI가 못할거야"라고 생각했었는데 반대였어요. 사람이 작성했던 것과 거의 유사하게 작성했습니다. 놀라울 정도로요.

AI에게 설명이 필요 없을 정도로 단순한 용어를 사용하자
프로젝트를 할 때에 특별한 관점이 담긴 개념을 생성해서 사용할 때가 많습니다. 사람들끼리 프로젝트를 할 때에는 효과적이지만 자칫하면 AI에게는 독이 될수도 있습니다. 오히려 확률적인 분포를 따르는 LLM 의 작동방식을 생각해봤을 때 단순한 용어, 단순한 개념, 단순한 아키텍처를 사용하는게 더 성능이 올라갈 수도 있습니다.

AI와 티키타카하면서 개선하자
에이전트가 잘하는 워크플로우를 만들기 위해서는 AI가 잘 설명할 수 있는 워크플로우를 선택하는게 좋습니다. 내 워크플로우를 LLM에게 보여주고 더 나은 방법을 제안해달라고 해보세요. 그리고 그 중에 괜찮은 방법을 선택해서 발전시켜달라고 해보세요. 이런 이터레이션이 돌아야 AI가 잘 이해하는 워크플로우가 될 가능성이 높습니다. 처음부터 끝까지 사람이 만든 워크플로우는 이해하기가 어려워요.

오늘은 Jake Park님과 같이 문제를 풀었습니다. 창업자이자 엔지니어 백그라운드를 가진 분과 대화할 수 있어서 즐거웠어요. 집요하게 문제를 끝까지 파고드는 쾌감을 느낄 수 있었습니다.
