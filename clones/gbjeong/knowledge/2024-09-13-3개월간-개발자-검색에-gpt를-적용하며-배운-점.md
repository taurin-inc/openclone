[3개월간 개발자 검색에 GPT를 적용하면서 배운 점]
- 크레딧 찾아서 받자. 예상치 못하게 API 사용량이 올라갈 때가 있다. OpenAI $2500크레딧은 우선 받고 시작하자.
- AI가 필요할 때 ChatGPT로 이동하고, 다시 어드민으로 돌아가느니, 한 번에 처리하는 시스템을 구축하는 게 훨씬 생산적이다.
- RAG는 필수다. Hybrid Search, Semantic 단위 chunking, 자동 태깅 값으로 필터 같은 포인트를 추가할 때마다 10배, 100배 breakthrough 를 체감했다.

‘쇼핑몰 PG 변경 업무를 맡을 Nodejs 개발자. 사용자 유입 경로에 따라 각기 다른 토스페이먼츠 모듈을 띄워야 함. 풀스택 우대’와 같은 공고를 입력하면 AI가 필터를 대신 입력하고 관련된 경험이 있는 크루부터 보여줍니다.
관련된 정보를 하이라이트하고, 추천 이유를 제시합니다. 마지막으로 미팅 제안 메시지까지 자동으로 생성하면 사람은 저장 버튼만 누르면 됩니다. 덕분에 더 높은 퀄리티로 더 많은 추천을 할 수 있었습니다.

OpenAI 크레딧 덕분에 비용 부담 없이 최적화를 시도할 수 있었습니다. 15배나 더 비싼 GPT-4o로 벤치마크를 만든 후에 mini로 줄이는 식으로요. Microsoft Founders Hub에서 크레딧을 신청할 수 있습니다. 다들 크레딧 받으시고 업무에 적극 활용하시길!
