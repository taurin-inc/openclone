제가 클로드코드를 잘 쓰게 된 계기는 불가능해보이는 개발 일정을 받았을 때였습니다. 3일만에 71개의 태스크를 완료하여 배포한 결과물을 고객사 앞에서 발표까지했습니다. 이렇게 AI를 안 쓰면 달성이 불가능한 환경을 만들면 잘 쓰게 됩니다.

첫 날, 4시간 잠
3시간 동안 38개의 요구사항을 받아적고 약 50페이지의 PPT, 20개의 기존 제품 가이드 동영상과 30개의 pdf 를 받았습니다. 첫 날에는 바로 개발을 하지 않고 클로드코드와 함께 요구사항을 정리했습니다. 모호한 요구사항을 클로드가 찾아내면 맥락을 추가해줬어요. 무엇이 명확한 요구사항인지 정의하는 작업이 필요했고, 만들어야 하는 제품의 도메인을 익혔습니다.

둘째 날, 3시간 잠
태스크를 등록하고 개발을 시작했습니다. Linear mcp 를 연결해서 모든 작업을 리스팅해서 중복, Parent-sub issue 를 등록했습니다. 약 50여개 이슈가 생성됐습니다. 하나씩 하나씩 이슈를 처리해서는 절대 달성이 불가능했습니다. Claude code 공식 문서를 통해 git worktree 를 발견하고 병렬로 실행하기 시작합니다.

셋째 날, 2시간 잠
프로젝트가 진행될수록 새로운 이슈가 생깁니다. 처음 이슈를 만들 때 놓쳤던 걸 추가하고, 클라이언트로부터 요구사항이 추가로 도착합니다. 간절함 덕분에 Multi Clauding 하는 세션의 개수를 늘어납니다. 하지만 세션 리밋에 도달하면 세션을 줄이고 api 로 사용합니다. 리밋이 풀리면 다시 세션을 왕창 늘려서 개발합니다. 서브에이전트, 슬래쉬 커맨드, 훅 필요한 건 다 씁니다.

이렇게 만들어서 발표한 결과물은 폭발적인 반응을 얻었습니다. 우리가 타겟했던 부서 뿐만 아니라 모든 부서에서 저희에게 일을 맡기고 싶다고 찾아오셨어요. 결국 모든 부서와 돌아가면서 미팅을 했습니다.

생산성을 높이고 싶다면 직접 코딩해서는 절대 달성하지 못할 목표를 세워보세요. 그리고 간절하게 지켜보세요. 분명히 얻어갈 수 있는 게 있을겁니다.
