스킬은 직접 만드는 Micro SaaS입니다. 에이전트 만들지 말고 Skills을 만드세요. 정교한 Skills가 많이 만들어지고 있습니다. 어떤 사람들은 Skills는 마크다운 패키징에 불과하다고 하지만, 이건 AI를 잘 쓰는 사람들의 패턴입니다.

1. 에이전트는 천재지만 전문가가 아닙니다
현재의 LLM 에이전트는 IQ 300의 천재처럼 복잡한 계산을 할 수 있지만, 세금 코드 같은 실무 지식은 매번 처음부터 파악해야 합니다. 필요한 건 경험 많은 전문가의 일관된 실행인데, 에이전트는 전문 지식을 잘 흡수하지 못하고 시간이 지나도 스스로 학습하지 못합니다.

2. 스킬은 그냥 폴더입니다
스킬은 에이전트를 위한 절차적 지식을 패키징한 파일 컬렉션입니다. 놀랍게도 구현은 그냥 폴더입니다. Git으로 버전 관리하거나, Google Drive에 저장하거나, 압축해서 공유할 수 있습니다. 이 단순함이 누구나 쉽게 만들고 사용할 수 있게 합니다.

3. 기존 툴과 다른 점
전통적인 툴은 설명서가 모호하고, 모델이 코드를 수정할 수 없고, 항상 컨텍스트 창에 상주합니다. 스킬은 코드가 스스로 문서화되고, 모델이 코드를 수정할 수 있고, 파일 시스템에 저장되어 있다가 필요할 때만 로드됩니다. 에이전트에게는 스킬의 메타데이터만 먼저 보여주고, 사용을 결정하면 그때 skill.md를 읽어서 전체 폴더 구조에 접근합니다.

4. 스킬의 세 가지 유형
- 기초 스킬: 전문적인 문서 작성, 과학 연구 능력 같은 새로운 기능을 부여합니다.
- 제3자 스킬: Browserbase의 Stagehand를 사용한 웹 탐색, Notion 작업 공간 이해처럼 파트너 소프트웨어를 더 잘 활용하게 합니다.
- 엔터프라이즈 스킬: 가장 큰 반응을 얻고 있습니다. 조직의 모범 사례나 내부 소프트웨어 사용법 같은 팀 고유의 지식을 에이전트에게 교육합니다.

5. 에이전트 아키텍처가 수렴하고 있습니다
에이전트 루프(모델의 내부 컨텍스트 관리), 런타임 환경(파일 시스템 및 코드 읽기/쓰기), MCP 서버(외부 세계의 툴 및 데이터 연결), 스킬 라이브러리(수백 개의 스킬 중 필요한 것만 실행 시점에 로드). 결국 에이전트에 새로운 기능을 부여하는 것은 적절한 MCP 서버와 스킬 라이브러리를 갖추는 것으로 귀결됩니다.

6. 30일 후의 Claude가 첫날보다 훨씬 유능해지는 것
스킬은 조직 내의 집단적이고 진화하는 지식 기반을 제공합니다. Claude가 작성한 절차적 지식은 표준화된 포맷을 통해 미래 버전의 자신에 의해 효율적으로 사용될 수 있어서 학습의 전이가 가능해집니다. Claude는 필요에 따라 새로운 기능을 즉시 습득하고, 발전시키며, 쓸모없어진 것은 버릴 수 있습니다.

7. 컴퓨터 공학에 비유하면
모델은 프로세서입니다. 거대한 잠재력을 가지지만 그 자체로는 한계가 있습니다. 에이전트 런타임은 운영 체제입니다. 프로세스, 자원, 데이터를 조율해서 프로세서의 가치를 극대화합니다. 스킬은 소프트웨어입니다. 수백만 명의 개발자가 도메인 전문 지식과 고유한 관점을 인코딩하는 계층입니다. 스킬은 이 마지막 계층을 모두에게 개방해서, 폴더에 무언가를 넣는 것만으로도 구체적인 문제를 해결할 수 있게 합니다.
