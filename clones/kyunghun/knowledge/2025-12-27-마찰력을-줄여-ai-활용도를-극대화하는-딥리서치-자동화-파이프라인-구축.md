어떻게 하면 AI를 더 자주 쓸 수 있을까요?

AI가 좋다는 것은 이제 누구나 다 압니다. 문제는 잘 안 쓰게 된다는 겁니다. 저도 흥미로운 스타트업을 발견하면 메모장에 적어둡니다. 나중에 GPT로 조사해야지. 그렇게 쌓인 리스트가 수십 개입니다. 실제로 조사한 건 몇 개 안 됩니다. 가끔 시간 내서 깊이 파보면 인사이트가 많은데, 꾸준히 못 합니다.

왜 그럴까요? GPT를 켜야 하고, 질문을 던지고, 답변을 보고, 부족하면 다시 물어봐야 합니다. 하나하나는 작은 허들인데, 이게 쌓이면 안 하게 됩니다. AI의 성능이 문제가 아닙니다. 시작하는 순간의 마찰이 문제입니다.

그래서 시작점을 바꿔봤습니다. URL 공유 한 번으로 리서치가 완성되는 파이프라인을 만들었습니다. 핵심은 두 가지였습니다. 아이폰 단축어로 트리거를 극한까지 낮춘 것, 그리고 Firecrawl로 한 번의 API 콜에 딥리서치 수준의 결과물을 얻은 것.

1. 아이폰 단축어가 웹훅을 보냅니다

아이폰 단축어 하면 보통 앱 열기, 메시지 보내기 정도를 떠올립니다. 하지만 실제로는 훨씬 많은 것이 가능합니다. 공유 시트에서 URL을 받아서 외부 서버로 POST 요청을 보낼 수 있습니다. n8n, Zapier, Make 같은 자동화 도구의 웹훅으로 직접 연결됩니다. 별도 앱 설치 없이, 애플이 기본 제공하는 기능만으로 가능합니다.

설정도 간단합니다. 단축어 앱에서 "URL 가져오기" → "URL의 콘텐츠 가져오기" (POST 방식) → 웹훅 URL 입력. 5분이면 끝납니다. 기사 보다가 공유 버튼 누르고, 단축어 선택하면 끝입니다. 새로운 행동을 만드는 게 아니라 이미 습관화된 동작에 기능을 얹는 것 입니다. 출퇴근길에 뉴스 보다가 흥미로운 회사 발견하면 공유 버튼 한 번. 회사 도착하면 노션에 리서치가 정리되어 있습니다.

2. 한 번의 API로 딥리서치가 가능해졌습니다

트리거를 해결했으면 다음은 리서치 품질입니다. 이전에 겪은 시행착오가 있습니다. 기업 조사를 할 때 다양한 방법을 시도했었습니다. Perplexity API도 써보고 MCP로도 해봤습니다. 문제가 많았습니다. 토큰 소모가 심하고, 여러 소스를 종합하는 과정에서 정리가 안 되고, 할루시네이션도 발생했습니다. 한 번의 API 콜로 깊이 있는 리서치를 얻는 게 생각보다 어려웠습니다.

드디어 Google Deep Research API가 나왔다고 해서 써보려 했는데 티어 문제인지 쿼터가 부족했습니다. 대안으로 찾은 게 Firecrawl입니다. 웹 데이터를 AI용으로 변환하는 API를 제공하는 스타트업인데 여기서 최근 출시한 Deep Research API가 괜찮았습니다. 단순 크롤링이 아니라 에이전트가 알아서 여러 소스를 탐색하고 교차 검증합니다. 한 5분 정도 걸리는데, 결과물이 웬만한 딥리서치 수준입니다. 아웃풋이 JSON이나 Markdown으로 구조화되어 나온다는 점도 좋았습니다. LLM에 바로 투입하거나 노션에 저장하기 좋은 형태입니다.

3. 허들을 낮추면 행동이 바뀝니다

최종 파이프라인은 이렇습니다. “아이폰 공유 버튼 → n8n 웹훅 → Firecrawl Agent 리서치 → AI 에이전트 보고서 정리 → 노션 DB 저장 → 팀 채팅 공유” 이제 리스트가 쌓이지 않고 바로바로 리서치가 됩니다.

만들면서 생각해봤습니다. AI가 좋다는 건 다 압니다. 그런데 왜 안 쓸까요? 결국 처음 시작하는 순간의 마찰이 의외로 큰 허들이었습니다. GPT 켜는 것, 프롬프트 쓰는 것, 결과 확인하고 다시 물어보는 것. 하나하나는 작은 허들인데, 이게 쌓이면 안 하게 됩니다. 허들을 낮추면 쉽게 씁니다. 쉽게 쓰면 자주 씁니다. 자주 쓰면 쌓입니다.

메모장에 URL을 적어두는 사람과 공유 버튼을 누르는 사람. 행동의 차이는 작지만, 1년 뒤 쌓인 결과물의 차이는 크다고 생각합니다.
