요즘 핫하다는 Claude Code로 글쓰는 에이전트를 만들어봤습니다

저는 Claude와 함께 글을 씁니다. 보통 1~2시간 정도 걸립니다. 주제를 던지고, 초안을 받고, "여기는 이렇게 바꿔줘", "이 표현 별로야", "더 구체적으로" 하며 티키타카를 합니다. 공부하려고 시작한 습관인데, 어느새 루틴이 됐습니다. 그런데 비효율이 계속 느껴졌습니다. 비슷한 피드백을 매번 반복하고 있었습니다. 분명히 말했는데 다음 세션에서는 리셋입니다. 또 처음부터 설명해야 합니다.

Claude 프로젝트를 만들어봤습니다. 제 스타일을 정리해서 지침으로, 과거의 글들을 지식으로 넣어뒀습니다. 꽤나 편해졌습니다. 그런데 뭔가 부족했습니다. 루프가 안 돌았습니다. 제가 피드백을 줘도 그게 자동으로 쌓이지 않았습니다. 지식을 제가 직접 업데이트해야 했습니다. 번거로워서 안 하게 됩니다. 결국 시스템이 성장하지 않습니다. 이걸 해결해보고 싶었습니다.

그래서 Claude Code로 직접 만들어봤습니다. 제 글 30개를 학습시키고, 그 스타일대로 글을 쓰고, 피드백이 자동으로 쌓이는 시스템입니다.

1. Claude Code가 쓰기 쉬워졌습니다

이전에는 Claude Code를 쓰려면 터미널이나 VS Code등에서 사용해야 했습니다. 딱히 불편한 건 아닌데, 비개발자에게는 심리적 허들이 있었습니다. "코딩 툴"이라는 느낌이 들면 일단 멈칫하게 됩니다. 그런데 이제는 그냥 Claude 앱에서 됩니다. 채팅 옆에 "코드"라는 항목이 생겼습니다. 클릭하고 원하는 걸 말하면 됩니다. 코딩하는 느낌이 아닙니다. 그냥 대화입니다. 다만 이 대화가 파일을 읽고, 쓰고, 수정할 수 있습니다. 그래서 피드백이 파일에 쌓이고, 세션이 바뀌어도 학습이 이어집니다. 일반 Claude와의 결정적 차이입니다.

2. 에이전트는 4개나 필요했습니다

처음에는 하나로 다 하려고 했습니다. "내 글 분석하고, 글 쓰고, 검토하고, 학습까지 해줘." 금방 한계가 보였습니다. 사람도 마찬가지입니다. 글 쓰는 사람이 동시에 자기 글을 객관적으로 평가하기 어렵습니다. 그래서 역할을 나눴습니다.

- Style Analyzer: 제 글을 분석합니다. 톤, 구조, 자주 쓰는 표현, 패턴까지 추출해서 스타일 가이드를 만듭니다. 새 글이 승인되면 이것도 분석해서 가이드를 업데이트합니다. 이 에이전트도 계속 성장합니다.
- Content Writer: 스타일 가이드만 보고 글을 씁니다. 분석은 Analyzer가 해줬으니, Writer는 글 쓰는 것에만 집중합니다. 역할이 명확하니 퀄리티가 올라갑니다.
- Content Reviewer: Writer가 쓴 글을 검토합니다. 본인이 쓴 글을 본인이 회고하면 객관성이 떨어집니다. 다른 에이전트가 보니까 "여기 톤이 스타일 가이드랑 다른데", "이 표현은 피드백 로그에서 싫다고 했던 건데" 같은 지적이 정확해집니다. 80점 미만이면 Writer에게 수정 지시를 보냅니다. 에이전트끼리 티키타카가 됩니다.
- Style Learner: 제가 "이건 내 스타일 아니야" 또는 "이거 좋아!"라고 하면, 이걸 해석해서 스타일 가이드를 업데이트합니다. 승인한 글은 새로운 학습 데이터로 저장됩니다.

핵심은 이 4개가 루프를 돈다는 겁니다. Writer가 글을 쓰면 → Reviewer가 검토하고 → 통과하면 제게 보여주고 → 제 피드백을 Learner가 받아서 → Analyzer의 스타일 가이드를 업데이트합니다. 다음에 Writer가 글을 쓸 때는 업데이트된 가이드를 참조합니다. 피드백이 휘발되지 않고 시스템에 쌓입니다. 쓸수록 정확해지는 구조입니다. 그리고, 이 4개를 하나의 "스킬"로 묶었습니다. 특정 작업을 위한 에이전트 팀입니다. "LinkedIn 글 써줘"라고 하면 이 스킬이 활성화되고, 4개의 에이전트가 알아서 협업합니다.

3. 모바일에서도 됩니다

매번 컴퓨터 켜기는 번거롭습니다. 출퇴근길에 글감이 떠올랐을 때 바로 쓰고 싶었습니다. 방법이 있었습니다. GitHub에 올리는 것 입니다. 스타일 가이드, 피드백 로그, 에이전트 설정 파일을 모두 GitHub 저장소에 넣었습니다. Claude 모바일 앱에서 이 저장소를 연결하면, 모바일에서도 동일한 시스템을 쓸 수 있습니다. 모바일에서 피드백한 내용이 그대로 GitHub에 반영됩니다. 모바일에서 "이 표현 싫어"라고 하면 스타일 가이드가 업데이트되고, 나중에 맥북에서 열어도 그 학습이 이어집니다. 디바이스가 바뀌어도 루프가 끊기지 않습니다.

아직 몇 번밖에 안 써봤습니다. 과연 얼마나 좋아질지는 더 실험해봐야 합니다. 그래도 왠지 든든합니다. 내 스타일을 분석하는 인턴, 글 쓰는 인턴, 검토하는 인턴, 피드백 정리하는 인턴. 4명이 한 팀으로 붙은 느낌입니다.
