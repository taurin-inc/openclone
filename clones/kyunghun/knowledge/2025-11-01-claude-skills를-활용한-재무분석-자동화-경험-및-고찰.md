Claude Skill을 활용하여 재무분석을 해보았습니다.

백문이 불여일행, AI는 직접 써봐야합니다. 요즘 바쁘다는 핑계로 AI에 대한 정보를 접하기만하고, 직접 써보지를 못했습니다. 화창한 주말에 그렇게 좋다는 클로드 스킬을 사용해보았습니다. 각종 분석툴을 제공한다는 Claude for Financial Services를 써보고 싶었지만 아직 Waitlist라서, 직접 만들어보기로 했습니다.

결과는, 12개월 손익계산서를 분석하는 자동화 시스템을 만드는 데 걸린 시간은 단 15분 정도였습니다. 접대비에 의도적으로 넣어둔 이상치를 탐지하고 상세 내역까지 분석한 보고서의 정확도도 완벽했습니다. 코딩 지식은 전혀 필요 없었습니다.

1. Claude Skills은 무엇일까요?

특정 업무를 자동화하기 위한 '지침 패키지'입니다. 재무 분석, 문서 작성, 데이터 검증 같은 반복 작업의 처리 방식을 스킬로 만들어두면, 클로드가 필요할 때 해당 스킬을 불러와 실행합니다. 마치 신입사원에게 업무 매뉴얼을 주는 것과 비슷합니다. 하지만 매뉴얼을 만드는 게 어렵지는 않을까요?

'skill-creator'라는 스킬이 있습니다. 스킬을 만들어주는 스킬입니다. 설치도 간단합니다. 자연어로 ‘skill-creator’ 설치해줘라고 말하면 끝입니다. 설치가 되면, "월별 손익계산서를 분석하는 스킬을 만들고 싶어”라는 명령어만 주면 됩니다. 5분정도 지나자, 400줄 정도의 체계적인 프롬프트로 구성된 스킬이 자동 생성되었습니다.

2. 기존 방식들과 무엇이 다를까요?

코딩을 몰라도 비슷한 작업을 하는 방법은 여러 가지가 있습니다.
- Cursor로 바이브코딩으로 엑셀 파일 분석
- MCP를 에이전트에 연결
- n8n이나 Make 같은 워크플로우 도구 사용

그런데 Claude Skills는 접근 방식이 약간 다릅니다.

1) 토큰 효율성: MCP는 모든 컨텍스트를 계속 로드해 토큰을 대량 소모합니다. Claude Skills는 YAML 메타데이터만 먼저 읽고, 필요할 때 해당 스킬만 불러옵니다. 수백 줄의 프로젝트 컨텍스트로 감당이 안 되던 것도, 스킬로 나누면 명확하고 효율적입니다. 토큰 소모량이 대폭 감소하였습니다.

2) 인터페이스 패러다임: n8n이나 Make는 워크플로우를 '다이어그램'으로 만듭니다. 노드를 연결하고, 조건을 시각화합니다. 직관적으로 보이지만, 복잡해지면 한계가 옵니다. 코드 노드가 필요해지고, 다이어그램도 매우 복잡해집니다. 그런데 생각해보면 코딩도 다이어그램으로 하지 않습니다. 그냥 텍스트로 합니다. 어쩌면 워크플로우도 마찬가지일지 모릅니다. 다이어그램보다 자연어로 설명하는 게 더 직관적이고 유연할 수 있습니다. 최근 Intercom이 비슷한 방향의 제품을 발표한 것도 이런 맥락으로 보입니다. Claude Skills는 기능별로 스킬만 만들어두면, 에이전트가 상황에 맞게 적절한 스킬을 선택해 조합합니다. 다이어그램의 시각적 직관성을 포기하는 대신, 자연어의 표현력과 유연성을 얻는 것입니다.

3) 모듈화된 구조와 코드 실행 통합: 폴더 기반 모듈화로, 각 스킬이 SKILL.md(지침), 스크립트, 리소스(예: 템플릿)를 포함한 독립 폴더로 구성됩니다. 이 덕분에 Claude가 작업 중 스킬을 자동 스캔하고, 필요 시 온디맨드 로드해 스택처럼 조합할 수 있습니다. 여기에 Code Execution Tool 연동으로 보안 환경에서 코드를 실제 실행하여, LLM의 고질적 문제인 복잡 계산(엑셀 공식)을 매우 정확하게 하고, 파일 생성(PowerPoint)을 토큰 생성의 불안정 없이 안정적으로 처리합니다.

3. 본질적인 변화는 '누가 만드는가'입니다

조직에서 AI 도입이 어려운 진짜 이유는 무엇일까요? 도메인을 가장 잘 아는 사람들이 AI까지 할 여력이 없기 때문입니다. AI가 아직 어렵기도 합니다. 하지만, 재무팀은 예산 분석의 맥락을 압니다. 어떤 계정이 민감한지, 어떤 패턴이 이상한지 압니다. 이 지식을 자동화하려면 개발팀에 요청해야 하지만, 우선순위에서 밀리고, 결국 개발되지 않습니다. Claude Skills는 이 구조를 바꿉니다. 재무팀이 직접 만들 수 있습니다. 자연어로 설명하면 됩니다. 수정도 자연어로 합니다. 도메인 전문성과 AI 활용 능력이 하나로 합쳐질 수 있습니다.

각 팀이 쌓아온 암묵지를 스킬로 명시화하면, 그것이 조직의 새로운 자산이 됩니다. 신입이 오면 스킬을 공유하고, 팀이 해체되어도 지식이 남습니다. 변화하는 규정과 프로세스를 스킬에 반영하는 것도 빠릅니다. 더 근본적으로는, '도구를 만드는 사람'과 '도구를 쓰는 사람'의 경계가 흐려집니다. 모든 지식 노동자가 자신의 영역에서 AI를 프로그래밍하는 시대가 오고 있습니다.

아직 시작 단계이지만, Claude Skills는 그 가능성을 충분히 보여주고 있다고 생각합니다.
