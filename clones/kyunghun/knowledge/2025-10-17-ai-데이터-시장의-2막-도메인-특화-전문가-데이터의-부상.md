AI 데이터 시장의 2막이 열립니다

작년에 OpenAI 공동창업자 Ilya Sutskever는 “우리는 peak data에 도달했다.” 라고 했습니다. 인터넷의 유용한 데이터를 이미 대부분 소진했다는 것이었습니다.

하지만 1년이 지난 지금, AI 회사들은 데이터 부족보다는 새로운 종류의 데이터 확보에 집중하고 있습니다. 스케일링 법칙이 끝났다라기 보다는 스케일링 법칙이 진화하고 있습니다.

1. 인터넷 데이터는 고갈됐다. 그런데 지식은?

물론, 인터넷의 텍스트, 코드, 이미지는 한계에 다다른 것일 수 있습니다. 하지만 모든 지식이 인터넷에 있는 것은 아닙니다. 투자은행가가 M&A 모델을 만드는 방법, 변호사가 계약서를 검토하는 과정, 의사가 진단을 내리는 방식. 이런 “머릿속 지식”은 문서화되지 않은 채 전문가들의 경험 속에만 존재합니다. 이걸 “PhD 레벨 데이터” 또는 “도메인 특화 전문가 데이터”라고 부릅니다.

이것이 AI의 다음 스케일링 법칙이 되고 있습니다.

2. 왜 코딩 AI가 먼저 성공했을까?

GitHub Copilot이나 Cursor 같은 코딩 도구들이 먼저 실용화된 데는 이유가 있습니다. 코드는 특별한 속성을 가지고 있습니다. 바로 ”명확한 정답 검증“ 입니다. 코드가 컴파일되는가? 유닛 테스트를 통과하는가? 올바른 API 응답을 반환하는가? 모두 명확하게 “맞다” 또는 “틀리다”로 판정할 수 있습니다. 게다가 GitHub 같은 오픈소스 저장소에 이미 방대한 코드 데이터가 존재했습니다.

반면 의료, 법률, 금융은 다릅니다. 치료가 효과가 있었는지는 알 수 있지만 왜 효과가 있었는지는 불명확하기도 합니다. 두 변호사가 다르게 추론해도 둘 다 타당할 수 있습니다. 거래로 수익을 냈지만 그게 분석력인지 타이밍인지 운인지 구분하기 어렵습니다. 이 차이가 바로 “다음 세대 데이터”의 핵심이지 않을까요. 그리고 단순한 텍스트가 아니라 추론 과정, 맥락, 근거를 담은 “멀티턴 데이터”가 필요합니다.

3. AI 회사들이 투자은행가를 찾는 이유

최근 흥미로운 변화가 있습니다. 주니어 투자은행가나 사모펀드 직원들을 리크루터들이 이제 AI 회사로 연결하고 있다고 합니다. 이유는 그들의 도메인 지식을 AI 모델 학습에 활용하기 위해서입니다. 아이러니하게도 우리는 AI가 우리 일을 대체할 수 있도록 데이터를 제공하고 있는 셈입니다.

파트타임 “데이터 레이블링” 시장이 성장하고 있습니다. 하지만 1세대 데이터 레이블링(“이게 고양이야 개야?”)과는 성격이 다릅니다. “변호사가 계약서를 검토하는 과정”을 어떻게 데이터화할까요? 화면을 녹화해야 할까요? 변호사가 생각을 말로 설명해야 할까요?

4. 어떤 문제를 풀어야 하는가

코드 데이터와 도메인 전문가 데이터의 근본적 차이는 “검증 가능성”입니다. Scale AI 같은 1세대 데이터 회사들은 “이미 정답이 명확한 것”을 레이블링했습니다. 하지만 “변호사 두 명이 같은 계약서를 다르게 검토해도 둘 다 합리적일 수 있다”면 어떻게 해야 할까요?

데이터 수집과 검증이 동시에 수반되어야 됩니다. 변호사가 계약서를 검토하는 과정을 녹화하는 것만으로는 부족합니다. 왜 그렇게 판단했는지, 어떤 선례를 참고했는지, 다른 접근은 없었는지까지 함께 캡처해야 할 것 입니다.

더 복잡한 것은 이 데이터가 “맞다/틀리다”로 판정되지 않는다는 겁니다. 여러 전문가의 다양한 접근을 모두 담아내면서도, 그중 어떤 것이 더 나은 결과로 이어졌는지를 추적해야 합니다. 의료 데이터라면 치료 결과와 연결하고, 법률 데이터라면 소송 결과나 협상 성과와 연결하고, 투자 데이터라면 실제 수익률과 연결해야 됩니다.

결국, 도메인에 대한 깊은 이해, 전문가 네트워크에 대한 접근성, 그리고 “과정과 결과를 연결하는 추적 시스템”이 필요합니다. 단순히 데이터를 모으는 게 아니라 데이터의 맥락과 결과를 함께 구조화하는 역량이 매우 중요합니다. AI 모델만이 아니라 이러한 인프라를 가지는 것 또한 정말 강력한 해자가 될 것이라고 생각합니다.
