AI 시대, 스타트업이 살아남기 위한 사고의 틀

정말 하루하루가 다이나믹한 세상입니다. 6개월 전 불가능했던 것이 오늘은 가능하고, 시장을 뒤흔들 빅테크의 제품이 쏟아지고 있습니다.

그런 혼란 속에서, 업계 전문가들이 여러 조언을 해주고 있습니다. 이 분들의 조언을 프레임워크로 정리를 해보았습니다. 물론 완벽한 답은 아닙니다. 하지만 적어도 어디서 싸울지, 어떻게 방어할지, 어떻게 죽지 않을지에 대한 사고의 틀은 가지면 좋을 것 같습니다.

조금이라도 도움이 되시길 바랍니다.
출처는 댓글로 달아두었습니다.

━━
Layer 1: 전략적 포지셔닝 (Where to Play)
"어디서 싸울 것인가?"

1.1 시장 선택: Vertical의 중요성
- ChatGPT는 모든 것을 잘하려고 합니다
- 우리는 하나를 완벽하게 해야 합니다
- 규제 산업(의료/법률/금융)이 방어하기 쉬습니다

1.2 제품 전략: Wrapper만으로는 24개월 안에 죽습니다
- OpenAI는 우리보다 똑똑하고, 빠르고, 자본도 많습니다
- API 위에 UI만 얹으면 6개월 내 복제됩니다
- 작은 문제 완벽히 해결 → 인접 문제로 확장 → 어느새 전체 워크플로우 장악
- 우리 제품이 사라지면 고객 업무가 멈추는 상태. 이게 Platform입니다

1.3 시간 지평: 오늘의 한계는 내일의 기회
- 2023년 "AI 코딩 불가능"이라며 포기한 자 vs 2025년 Cursor
- 차이는 2년전 선 베팅입니다
- 지금 GPT-5로 안 되는 것 = 2026년 GPT-6로 되는 것
- 미래 모델 전제로 설계 안 하면 이미 늦었습니다

━━
Layer 2: 경쟁 우위 구축 (How to Win)
"어떻게 방어할 것인가?"

2.1 자산 기반 Moat: 데이터와 네트워크
- 데이터 Moat = 사용할수록 똑똑해지는 루프입니다
- Network effect의 법칙: 사용자 1명 추가 = 기존 유저 모두 가치 상승
- 이게 없으면 가격 경쟁 지옥입니다
- Cursor: 10만 개발자가 코드 칠 때마다 학습 → 경쟁자는 10만 배 느린 속도

2.2 실행 기반 Moat: 복잡성과 속도
- Demo는 주말동안에도 만듭니다
- Production은 10,000개 edge case 처리입니다
- AI 시대는 iteration 주기가 분기 → 일 단위
- Early momentum이 카테고리 리더를 만듭니다
- 3개월 늦게 출시 = 시장의 70% 상실
- Early winner가 talent·capital·brand를 독점합니다

━━━
Layer 3: 실행 역량 (How to Execute)
"누가, 어떻게, 얼마나 빠르게?"

3.1 인재 선택 (Who): 배울 사람 vs 아는 사람
- 산업 경험보다 빠른 학습 능력과 질문하는 능력이 우선입니다
- Uber, Airbnb 창업자는 해당 산업 경험이 없었습니다
- First principles로 재창조했습니다
- 거대 조직보다 작은 조직으로 나누는게 필요합니다

3.2 실행 방식 (How): 판단 > 프로세스
- AI 시대는 3개월마다 변합니다
- 기존 가정에 의문 제기하는 문화가 혁신을 만듭니다
- AI로 생산성 10배 올렸다고 직원을 늘리면 안됩니다
- 스타트업이 죽는 이유: 프로세스는 늘고 판단은 줄어서입니다

3.3 변화 속도 (Speed): 잊는 속도 > 배우는 속도
- Khosla 질문: "3개월 전 당신 생각이 지금과 같다면?" → 이미 죽었습니다
- 경험 많은 팀이 신입보다 느린 진짜 이유: 배울 게 많아서가 아니라 잊어야 할 게 많아서입니다

━━━
Layer 4: 생존 기본 (How to Survive)
"어떻게 죽지 않을 것인가?"

4.1 재무: Default Alive가 전부입니다
- Default Alive = 수익이 먼저 vs Default Dead = 돈이 먼저 떨어짐
- 많은 창업자가 이 구분을 못하거나 안합니다
- 사용자 100만 ≠ 성공
- 사람들이 쓴다 ≠ 사람들이 돈 낸다
- Product-Market Fit ≠ Business-Market Fit
- "다음 투자 받으면 되지" = 가장 비싼 착각
- 돈 떨어지기 6개월 전부터 준비 안 하면 이미 죽습니다

4.2 실행: Customer > Technology
- 기술 완벽함은 0점. 고객 획득이 100점
- 순서를 바꾸면 망합니다

4.3 속도: 빠른 실패가 느린 성공을 이깁니다
- 2008 금융위기 = Airbnb·Uber 탄생
- 불황 = 경쟁자 적음 + 인재 구하기 쉬움 + 진짜 문제에 집중
- AI가 빠른 실패를 가능하게 합니다
- 3개월에 10번 시도 vs 3년에 1번 시도. 승자는 명확합니다
- 타이밍 탓하는 순간 진 것입니다. 시장이 아니라 실행에 베팅해야 합니다
