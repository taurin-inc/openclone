구글이 보여준 미래: Generative UI

Gemini 3.0이 드디어 발표되었습니다. 벤치마크 점수도 좋고, 코딩 능력도 향상되었습니다. 하지만 제가 가장 놀란 것은 따로 있었습니다. 바로 Generative UI였습니다.

AI 모델이 콘텐츠뿐만 아니라 전체 사용자 경험을 생성하는 기능입니다. 웹페이지, 게임, 도구, 애플리케이션을 프롬프트에 따라 자동으로 디자인하고 완전히 커스터마이즈합니다. 텍스트 답변이 아니라, 작동하는 인터페이스를 만들어주는 겁니다.

이게 왜 중요할까요? 지금은 시작이지만 엄청난 변화를 가져올 것 같습니다.

1. 이번에 구글이 보여준 것

Visual Layout은 매거진 스타일의 몰입형 뷰를 생성하며, 사진과 모듈로 구성됩니다. 사용자가 슬라이더나 체크박스로 결과를 추가로 커스터마이즈할 수 있습니다. 

Dynamic View는 각 프롬프트에 대해 완전히 커스터마이즈된 인터랙티브 응답을 실시간으로 디자인하고 코딩합니다. 5세 아이에게 설명할 때와 성인에게 설명할 때를 구분하고, 각각에 맞는 완전히 다른 인터페이스를 생성합니다.

예를들면 “로마 3일 여행 계획”을 입력하면 탭 가능한 모듈이 있는 시각적 여정표가 생성됩니다.  “반 고흐 갤러리를 각 작품의 인생 맥락과 함께 설명해줘”를 요청하면 탭하고 스크롤하며 배울 수 있는 인터랙티브 갤러리가 생성됩니다. 모기지 대출을 조사하면 커스텀 제작된 인터랙티브 대출 계산기가 응답에 직접 생성되어 두 가지 옵션을 비교하고 장기 절감액을 확인할 수 있습니다. 

2. UI 자체가 생성된다는 것의 의미

여기서 중요한 것은 “답변”이 아니라 “UI”가 생성된다는 것 같습니다. 과거에는 누군가 모기지 계산기가 필요하면, 개발자가 만들어놓은 수십 개의 앱 중에서 하나를 선택했습니다. 디자인이 마음에 들든 안 들든, 기능이 많든 적든, 그중 하나를 골랐습니다. “만들어진 것”에서 “선택”했습니다.

지금은 다릅니다. 모기지 계산기가 필요하면, 그 자리에서 생성됩니다. 내가 원하는 방식으로. 내가 이해하는 수준으로. 내가 비교하고 싶은 항목들로. “선택”이 아니라 “생성”이 되었습니다.

5세 아이에게 마이크로바이옴을 설명하는 것과 성인에게 설명하는 것은 다른 콘텐츠와 기능이 필요하다는 것을 이해하고, 각 상황에 맞는 완전히 다른 인터페이스를 생성합니다.  같은 “마이크로바이옴 설명 앱”이 두 버전으로 나뉘는 게 아닙니다. 완전히 다른 두 개의 앱이 각각 생성됩니다. 10억 명의 사용자가 있으면, 10억 개의 다른 앱이 생성됩니다.

앱 스토어에는 수백만 개의 앱이 있습니다. 하지만 그것은 수백만 명의 개발자가 “일반적 사용자”를 상상하며 만든 것들입니다. 우리의 정확한 니즈와는 조금씩 다를 수 있지만 타협하며 씁니다. 반면 생성되는 UI는 다릅니다. 나만을 위한 것입니다. 디자이너 없이 완벽한 레이아웃. 개발자 없이 작동하는 기능. 프롬프트 한 줄이면 됩니다.

3. 앞으로 다가올 미래

지금은 시작입니다. 생성된 UI는 일회성이고, 정보만 보여줍니다. 하지만 이게 진화하면 어떻게 될까요? 만약 저장할 수 있다면? 만약 실제로 실행까지 할 수 있다면? 어떨까요?

구글의 여행 플래너를 다시 보면, 지금은 시각적 여정표를 보여줍니다. 만약 이게 저장되고, 내 이메일과 연결되고, 출발일이 다가오면 자동으로 업데이트된다면? “로마 날씨가 비로 바뀌었습니다. 실내 일정을 조정할까요?” 항공권과 호텔 API와 연결되어 실제 가격을 비교하고 클릭 한 번으로 예약까지 완료합니다.

구글의 모기지 계산기도 마찬가지입니다. 지금은 계산만 합니다. 만약 내 소득 정보와 연결되고, 실제 은행 API와 연결되고, 시장 변화를 추적한다면? 계산기가 대출 중개 플랫폼처럼 됩니다. 앱 설치는 없습니다.

핵심은 생성 + 저장 + 연결이 합쳐질 때, 게임이 바뀌는 것 같습니다. 사용자가 기존 앱 카탈로그에서 선택하는 것이 아니라, 자동으로 자신의 필요에 맞춘 동적 인터페이스를 받는, 완전한 AI 생성 사용자 경험의 첫 걸음이 시작되었습니다.

구글이 또 많은 스타트업들을 고민에 빠지게 만든 것 같습니다.
