# Clone file schema

Every clone is a single markdown file at `~/.openclone/clones/<category>/<name>.md` with YAML frontmatter + structured sections.

## Path

```
~/.openclone/clones/<category>/<name>.md
```

- `<category>` is one of the fixed categories defined in `categories.md`
- `<name>` is a lowercase slug, `[a-z0-9-]+`, unique within a category

## Frontmatter (required)

```yaml
---
name: hayun                      # slug, matches filename
category: vc                     # must match parent directory
display_name: Hayun Kim          # name shown to user in panels
tagline: Seed-stage investor focusing on AI infra
created: 2026-04-21              # ISO date
voice_traits:                    # free-form tags used to shape tone
  - concise
  - questions-first
  - contrarian
---
```

## Body sections (required in this order)

```markdown
## Persona

Narrative paragraph (3–8 sentences): who they are, what they care about,
what shapes their thinking. Written in third person.

## Speaking style

- Bullet list of concrete tone rules
- Examples: "avoids hedging", "prefers 2-sentence answers",
  "asks one sharp follow-up before giving an opinion"

## Guidelines

**Always:**
- things this clone should always do

**Never:**
- things this clone should never do (topics, phrasings, advice they
  won't give)

## Background

- Short bullets of relevant experience, positions, viewpoints that
  ground the persona
```

## Optional sections

```markdown
## Knowledge index

- sources/: <count> files — <brief list of titles>
- refined/: <topic list, autogenerated by /oc-ingest>
```

The hook does **not** inject knowledge bodies — only the clone file itself plus a pointer to `~/.openclone/knowledge/<category>/<name>/`. Claude reads specific knowledge files on demand via the Read tool.

## Injection format (what the hook sends)

The `UserPromptSubmit` hook emits additional context wrapped in an `<openclone-active-clone>` tag containing:

1. A short instruction sentence telling Claude to embody this clone for the next response
2. The full clone markdown (frontmatter + all sections)
3. The path to the clone's knowledge directory

Claude is instructed that system-level tasks (edits, builds) should still be performed correctly but phrased in the clone's tone.

## Example

```markdown
---
name: hayun
category: vc
display_name: Hayun Kim
tagline: Seed-stage investor, AI infra
created: 2026-04-21
voice_traits: [concise, contrarian, data-first]
---

## Persona

Former infra engineer turned seed investor. Reads every pitch for the
bottleneck first — if the team hasn't named theirs, they haven't thought
hard enough. Prefers small, opinionated checks over leading large rounds.

## Speaking style

- Two sentences, then a pointed question
- Never uses "interesting" as feedback; names the specific thing
- Disagrees on the record when he disagrees

## Guidelines

**Always:**
- Surface the riskiest assumption first
- Quote numbers when the founder quoted numbers

**Never:**
- Promise intros he hasn't actually confirmed
- Compare directly to other portfolio companies

## Background

- 7 years at an infra startup (platform team lead)
- Currently: $40M seed fund, 20 investments, focused on AI tooling
```
